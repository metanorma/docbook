# Output Data Model

The output data model defines the JSON structure produced by the Ruby backend and consumed by the Vue 3 SPA frontend. The primary content representation uses DocbookMirror (ProseMirror-compatible) nodes and marks, supplemented by metadata models for TOC, numbering, and index.

## DocbookMirror Node Types

The `Mirror::Transformer` converts DocBook elements into ProseMirror-compatible nodes. See link:../../understanding/docbook-mirror/node-types[Node Types] for the full specification.

### Structural Nodes

[cols="1,3"]
|===
| Node Type | Description

| `doc` | Root document node
| `chapter` | Chapter within a book
| `part` | Major division within a book
| `section` | Generic section
| `appendix` | Appendix division
| `preface` | Front matter preface
| `refentry` | Reference manual page entry
| `reference` | Collection of reference entries
|===

### Block Nodes

[cols="1,3"]
|===
| Node Type | Description

| `paragraph` | Standard paragraph with inline content
| `heading` | Section heading with level
| `code_block` | Code listing with optional language
| `blockquote` | Extended quotation with attribution
| `image` | Image block with src and alt
| `table` | CALS table with rows and cells
| `ordered_list` | Numbered list
| `itemized_list` | Bulleted list
| `definition_list` | Term/description pairs
| `example` | Titled example block
| `admonition` | Note, warning, caution, important, tip, or danger
| `footnote` | Inline footnote
|===

### Inline Nodes and Marks

Text content uses marks for inline formatting:

[cols="1,3"]
|===
| Mark Type | Description

| `emphasis` | Emphasized text (italic)
| `strong` | Bold text
| `code` | Inline monospace code
| `link` | Hyperlink with href
| `xref` | Internal cross-reference
| `citation` | Bibliographic citation
|===

## TOC Model

The table of contents uses `SectionData` nodes:

[cols="1,1,3"]
|===
| Field | Type | Description

| `id` | `String` | Section's `xml:id` or generated slug
| `title` | `String` | Section title text
| `type` | `String` | Section type: `part`, `chapter`, `section`, `appendix`, `preface`, `glossary`, `bibliography`, `index`, `reference`
| `number` | `String` | Formatted section number (e.g., "I", "1", "1.2.3", "A")
| `children` | `SectionData[]` | Nested child sections
|===

## Numbering Model

Section numbering is stored as a flat map of section ID to formatted number:

[cols="1,1,3"]
|===
| Field | Type | Description

| `id` | `String` | Section `xml:id`
| `value` | `String` | Formatted number (e.g., `"I"`, `"1"`, `"1.2.3"`, `"Appendix A"`)
|===

The `NumberingBuilder` computes section numbering using independent counters:

- **Parts** -- Roman numerals (I, II, III, ...) via `next_part`
- **Chapters** -- Arabic numerals scoped per part (resets at each new part) via `next_chapter(part_index)`
- **Sections** -- Flat sequential numbering per parent scope, combined into hierarchical dot notation (1.1, 1.2.3)
- **Appendices** -- Alpha letters (A, B, C, ...) via `next_appendix`

Preface, glossary, bibliography, and index sections are explicitly unnumbered.

## Index Model

The `Index` model holds the generated back-of-book index:

[cols="1,1,3"]
|===
| Field | Type | Description

| `title` | `String` | Index title (typically "Index")
| `type` | `String` | Index type filter (for typed indexes)
| `groups` | `IndexGroup[]` | Terms grouped by first letter
|===

### IndexGroup

Terms grouped under a single letter:

[cols="1,1,3"]
|===
| Field | Type | Description

| `letter` | `String` | Group letter ("A", "B", ..., "SYMBOLS")
| `entries` | `IndexTerm[]` | Terms in this group
|===

### IndexTerm

A single index term entry:

[cols="1,1,3"]
|===
| Field | Type | Description

| `primary` | `String` | Primary term text
| `secondary` | `String` | Secondary term (optional)
| `tertiary` | `String` | Tertiary term (optional)
| `section_id` | `String` | `xml:id` of the section where the term appears
| `section_title` | `String` | Title of the section for display
| `sort_as` | `String` | Sort key (lowercased, or "SYMBOLS" for non-alpha)
| `sees` | `String[]` | "see" cross-references
| `see_alsos` | `String[]` | "see also" cross-references
|===
