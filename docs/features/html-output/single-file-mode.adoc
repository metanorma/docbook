# Single File Mode

The `build` command produces a single, self-contained HTML file with all assets inlined. This is the default and only output mode, ideal for distributing documents that can be opened directly in any browser, including via the `file://` protocol.

## How It Works

When building, the `Output::SinglePage` class inlines every asset directly into the HTML:

. **CSS** -- The compiled `app.css` from `frontend/dist/` is injected inline via a `<style>` tag
. **JavaScript** -- The compiled `app.iife.js` from `frontend/dist/` is injected inline via a `<script>` tag
. **Fonts** -- Google Fonts (Inter, Merriweather, JetBrains Mono) are loaded from CDN
. **DocbookMirror JSON** -- All document data (TOC, content, index) is serialized to DocbookMirror JSON and embedded as `window.DOCBOOK_DATA`

## Image Embedding

Images are embedded as base64 data URIs by default. The image strategy can be configured:

- `:data_url` -- Base64-encoded and embedded (default)
- `:file_url` -- Absolute file system paths
- `:relative` -- Relative paths for web hosting

The MIME type is detected automatically using the https://github.com/rails/marcel[Marcel] gem, which examines the file's magic bytes rather than relying on file extensions.

Image paths are resolved by searching the document's base path and its parent directories, trying common prefixes like `resources/` and `../resources/`. This accommodates the various directory structures used in DocBook projects.

## Data Injection

The DocbookMirror JSON data is embedded in the HTML:

[source,html]
----
<script>
window.DOCBOOK_DATA = { /* DocbookMirror JSON + TOC + numbering + index */ };
</script>
----

The Vue SPA's `documentStore` detects `window.DOCBOOK_DATA` and loads the data synchronously without any network requests. The `MirrorRenderer` component renders the DocbookMirror nodes into HTML.

## Use Case

Single file mode is designed for:

- **Document distribution** -- Send a single HTML file via email or file share
- **Offline reading** -- Works without a web server; open directly from the filesystem
- **Archival** -- Self-contained documents with no external dependencies (except CDN fonts)
- **Email attachments** -- One file to attach, no directory structures to manage

## CLI Usage

```bash
# Build an interactive reader
docbook build document.xml -o output.html

# With custom image paths
docbook build document.xml -o output.html --image-search-dir media/ images/

# For web hosting (external image files)
docbook build document.xml -o output.html --image-strategy relative
```

## Programmatic Usage

[source,ruby]
----
require "docbook"

Docbook::Output::SinglePage.new(
  xml_path: "book.xml",
  output_path: "output/book.html",
  image_search_dirs: ["media/", "images/"],
  image_strategy: :data_url,
  title: "My Book"
).generate
----
