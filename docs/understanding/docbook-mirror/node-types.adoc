# Node Types

Every element in a DocbookMirror JSON document is represented as a **node**.
Each node has a `type` string that identifies its kind, an optional `attrs`
object for type-specific attributes, an optional `content` array of child
nodes, and an optional `marks` array for inline annotations.

All node classes are defined in `Docbook::Mirror::Node` subclasses and are
registered in the `NODES` hash for deserialization.

== Common node shape

Every node follows this general structure when serialized to JSON:

[source,json]
----
{
  "type": "<node_type>",
  "attrs": { /* type-specific attributes */ },
  "content": [ /* child nodes */ ],
  "marks": [ /* mark objects, only on text nodes */ ]
}
----

* `type` (required) -- The node type string, matching the class's `PM_TYPE`
  constant.
* `attrs` (optional) -- A key-value map of attributes. Omitted when empty.
* `content` (optional) -- An ordered array of child nodes. Omitted when empty.
* `marks` (optional) -- An array of mark objects. Used only on `text` nodes.

== Document root

=== doc

`Docbook::Mirror::Node::Document` +
PM_TYPE: `"doc"`

The top-level container for the entire document. Produced by
`Transformer#from_docbook` as the entry point. The `title` attribute is
populated from the DocBook `info/title` element.

.Attributes

[cols="1,1,3"]
|===
| Attribute | Type | Description
| `title` | String | Document title, from `info/title`
|===

.Example
[source,json]
----
{
  "type": "doc",
  "attrs": {
    "title": "Metanorma Users Guide"
  },
  "content": [
    { "type": "chapter", "attrs": { "title": "Introduction" }, "content": [...] }
  ]
}
----

== Text

=== text

`Docbook::Mirror::Node::Text` +
PM_TYPE: `"text"`

A leaf node holding a run of inline text. Text nodes are the only node type
that carries a `text` attribute and may have `marks` applied to them. Text
nodes cannot have `content` children.

.Attributes

[cols="1,1,3"]
|===
| Attribute | Type | Description
| `text` | String | The text content of this run
|===

.Example (plain text)
[source,json]
----
{
  "type": "text",
  "text": "Hello, world."
}
----

.Example (text with an emphasis mark)
[source,json]
----
{
  "type": "text",
  "text": "important",
  "marks": [
    { "type": "emphasis" }
  ]
}
----

== Block nodes

=== paragraph

`Docbook::Mirror::Node::Paragraph` +
PM_TYPE: `"paragraph"`

A block-level paragraph. Mapped from DocBook `<para>` and `<formalpara>`.
Its `content` array contains inline content: text nodes, mark-annotated text,
and inline images.

.Example
[source,json]
----
{
  "type": "paragraph",
  "content": [
    { "type": "text", "text": "This is a " },
    { "type": "text", "text": "paragraph", "marks": [ { "type": "emphasis" } ] },
    { "type": "text", "text": " with inline markup." }
  ]
}
----

=== code_block

`Docbook::Mirror::Node::CodeBlock` +
PM_TYPE: `"code_block"`

A block of preformatted code or verbatim text. Mapped from DocBook `<code>`,
`<programlisting>`, `<screen>`, `<literallayout>`, `<example>`, and
`<informalexample>`.

.Attributes

[cols="1,1,3"]
|===
| Attribute | Type | Description
| `language` | String | Programming language for syntax highlighting
| `xml_id` | String | XML identifier, for cross-reference anchoring
| `title` | String | Title text (from `<example>` or `<figure>`)
| `callouts` | Array | Callout markers extracted from inline `<co>` elements (see below)
|===

.Callout markers

When a `<programlisting>` contains `<co>` (callout origin) elements, the
transformer extracts them as numbered markers. Each marker has:

[cols="1,1,3"]
|===
| Field | Type | Description
| `number` | Integer | Sequential marker number (1-based)
| `id` | String | The `xml:id` of the `<co>` element
| `label` | String | Display label (from `label` attribute, or the number)
|===

The code text has markers inserted as `(1)`, `(2)`, etc. at the `<co>`
positions. The `callouts` array in attrs provides the metadata for the frontend
to render styled badges and link to the corresponding `calloutlist`.

.Example (with callouts)
[source,json]
----
{
  "type": "code_block",
  "attrs": {
    "language": "xml",
    "callouts": [
      { "number": 1, "id": "co1", "label": "1" },
      { "number": 2, "id": "co2", "label": "2" }
    ]
  },
  "content": [
    { "type": "text", "text": "<book(1)>(2)\n  <title>Test</title>\n</book>" }
  ]
}
----

=== blockquote

`Docbook::Mirror::Node::Blockquote` +
PM_TYPE: `"blockquote"`

A block quotation. Mapped from DocBook `<blockquote>`. Contains block-level
children (paragraphs, lists, and so on).

.Example
[source,json]
----
{
  "type": "blockquote",
  "content": [
    {
      "type": "paragraph",
      "content": [
        { "type": "text", "text": "To be or not to be." }
      ]
    }
  ]
}
----

=== bullet_list

`Docbook::Mirror::Node::BulletList` +
PM_TYPE: `"bullet_list"`

An unordered (bulleted) list. Mapped from DocBook `<itemizedlist>`. Contains
`list_item` children.

.Example
[source,json]
----
{
  "type": "bullet_list",
  "content": [
    {
      "type": "list_item",
      "content": [
        {
          "type": "paragraph",
          "content": [ { "type": "text", "text": "First item" } ]
        }
      ]
    },
    {
      "type": "list_item",
      "content": [
        {
          "type": "paragraph",
          "content": [ { "type": "text", "text": "Second item" } ]
        }
      ]
    }
  ]
}
----

=== ordered_list

`Docbook::Mirror::Node::OrderedList` +
PM_TYPE: `"ordered_list"`

A numbered list. Mapped from DocBook `<orderedlist>`. Contains `list_item`
children. The structure is identical to `bullet_list` except for the type.

.Example
[source,json]
----
{
  "type": "ordered_list",
  "content": [
    {
      "type": "list_item",
      "content": [
        {
          "type": "paragraph",
          "content": [ { "type": "text", "text": "Step one" } ]
        }
      ]
    }
  ]
}
----

=== list_item

`Docbook::Mirror::Node::ListItem` +
PM_TYPE: `"list_item"`

A single list item inside a `bullet_list` or `ordered_list`. Mapped from
DocBook `<listitem>`. Contains block-level children (paragraphs, nested lists,
and so on).

.Example
[source,json]
----
{
  "type": "list_item",
  "content": [
    {
      "type": "paragraph",
      "content": [ { "type": "text", "text": "Item content" } ]
    }
  ]
}
----

=== dl

`Docbook::Mirror::Node::DefinitionList` +
PM_TYPE: `"dl"`

A definition list (variable list). Mapped from DocBook `<variablelist>`. Its
`content` array holds alternating `definition_term` and `definition_description`
nodes, paired to represent each `<varlistentry>`.

.Example
[source,json]
----
{
  "type": "dl",
  "content": [
    {
      "type": "definition_term",
      "content": [ { "type": "text", "text": "API" } ]
    },
    {
      "type": "definition_description",
      "content": [
        {
          "type": "paragraph",
          "content": [ { "type": "text", "text": "Application Programming Interface." } ]
        }
      ]
    }
  ]
}
----

=== definition_term

`Docbook::Mirror::Node::DefinitionTerm` +
PM_TYPE: `"definition_term"`

The term being defined in a definition list entry. Mapped from DocBook `<term>`.
Contains inline content (text nodes with optional marks).

.Example
[source,json]
----
{
  "type": "definition_term",
  "content": [
    { "type": "text", "text": "REST" }
  ]
}
----

=== definition_description

`Docbook::Mirror::Node::DefinitionDescription` +
PM_TYPE: `"definition_description"`

The definition or description associated with a term. Mapped from the
`<listitem>` inside a `<varlistentry>`. Contains block-level children.

.Example
[source,json]
----
{
  "type": "definition_description",
  "content": [
    {
      "type": "paragraph",
      "content": [ { "type": "text", "text": "Representational State Transfer." } ]
    }
  ]
}
----

== Headings

=== heading

`Docbook::Mirror::Node::Heading` +
PM_TYPE: `"heading"`

A section heading. Registered as a node type and available for heading-level
content. The heading level is expressed via attributes.

.Attributes

[cols="1,1,3"]
|===
| Attribute | Type | Description
| `level` | Integer | Heading level (1 through 6, mapping to h1-h6)
|===

.Example
[source,json]
----
{
  "type": "heading",
  "attrs": { "level": 1 },
  "content": [
    { "type": "text", "text": "Chapter Title" }
  ]
}
----

== Structural nodes

These nodes represent the hierarchical structure of a DocBook document.
DocBook has a richer structural vocabulary than typical web documents, and
DocbookMirror preserves this.

=== chapter

`Docbook::Mirror::Node::Chapter` +
PM_TYPE: `"chapter"`

A book chapter. Mapped from DocBook `<chapter>`.

.Attributes

[cols="1,1,3"]
|===
| Attribute | Type | Description
| `title` | String | Chapter title
| `xml_id` | String | XML identifier (`xml:id`), used for cross-references
| `number` | String | Computed chapter number (for example, `"1"`, `"2"`)
|===

.Example
[source,json]
----
{
  "type": "chapter",
  "attrs": {
    "title": "Getting Started",
    "xml_id": "getting-started",
    "number": "1"
  },
  "content": [
    { "type": "paragraph", "content": [ { "type": "text", "text": "In this chapter..." } ] }
  ]
}
----

=== section

`Docbook::Mirror::Node::Section` +
PM_TYPE: `"section"`

A generic section. Mapped from DocBook `<section>` and `<simplesect>`.

.Attributes

[cols="1,1,3"]
|===
| Attribute | Type | Description
| `title` | String | Section title
| `xml_id` | String | XML identifier
| `number` | String | Computed section number (for example, `"1.2"`)
|===

.Example
[source,json]
----
{
  "type": "section",
  "attrs": {
    "title": "Installation",
    "xml_id": "installation",
    "number": "1.1"
  },
  "content": [
    { "type": "paragraph", "content": [ { "type": "text", "text": "To install..." } ] }
  ]
}
----

=== reference

`Docbook::Mirror::Node::Reference` +
PM_TYPE: `"reference"`

A reference page collection. Mapped from DocBook `<reference>`.

.Attributes

[cols="1,1,3"]
|===
| Attribute | Type | Description

|===

.Example

=== refentry

`Docbook::Mirror::Node::RefEntry` +
PM_TYPE: `"refentry"`

A single reference entry (manual page). Mapped from DocBook `<refentry>`. The
transformer synthesizes a synopsis paragraph from `<refmeta>` data (field
signature, refentrytitle, manvolnum) when available.

.Attributes

[cols="1,1,3"]
|===
| Attribute | Type | Description
| `title` | String | Resolved entry title
| `xml_id` | String | XML identifier (from `refentry` or synthesized)
|===

.Example
[source,json]
----
{
  "type": "refentry",
  "attrs": {
    "title": "docbook(1)",
    "xml_id": "docbook-1"
  },
  "content": [
    {
      "type": "paragraph",
      "attrs": { "class": "refmeta-synopsis" },
      "content": [
        { "type": "text", "text": "String ", "marks": [ { "type": "code", "attrs": { "role": "type" } } ] },
        { "type": "text", "text": "name", "marks": [ { "type": "code", "attrs": { "role": "varname" } } ] }
      ]
    },
    { "type": "refsection", "attrs": { "title": "Description" }, "content": [...] }
  ]
}
----

=== refsection

`Docbook::Mirror::Node::RefSection` +
PM_TYPE: `"refsection"`

A section within a reference entry. Mapped from DocBook `<refsection>`,
`<refsect1>`, `<refsect2>`, and `<refsect3>`.

.Attributes

[cols="1,1,3"]
|===
| Attribute | Type | Description
| `title` | String | Section title
| `xml_id` | String | XML identifier
|===

.Example
[source,json]
----
{
  "type": "refsection",
  "attrs": {
    "title": "Description",
    "xml_id": "docbook-description"
  },
  "content": [
    { "type": "paragraph", "content": [ { "type": "text", "text": "Processes DocBook XML..." } ] }
  ]
}
----

== Media

=== image

`Docbook::Mirror::Node::Image` +
PM_TYPE: `"image"`

An image. Mapped from DocBook `<figure>`, `<informalfigure>`, and
`<inlinemediaobject>`. When a figure contains no actual image data but has an
`xml_id`, the transformer emits a placeholder paragraph instead so that the
cross-reference anchor is preserved.

.Attributes

[cols="1,1,3"]
|===
| Attribute | Type | Description
| `src` | String | Image file path (from `imagedata/fileref`)
| `alt` | String | Alternative text description
| `title` | String | Figure title
| `xml_id` | String | XML identifier for cross-referencing
|===

.Example
[source,json]
----
{
  "type": "image",
  "attrs": {
    "src": "images/architecture.png",
    "title": "System Architecture",
    "xml_id": "fig-architecture"
  }
}
----

== Tables

DocBook tables use a `tgroup` model with `thead` and `tbody`. The transformer
flattens this into a simpler `table` > `table_head` / `table_body` >
`table_row` > `table_cell` hierarchy.

=== table

`Docbook::Mirror::Node::Table` +
PM_TYPE: `"table"`

A table. Mapped from DocBook `<table>` and `<informaltable>`.

.Attributes

[cols="1,1,3"]
|===
| Attribute | Type | Description
| `xml_id` | String | XML identifier
| `title` | String | Table title
| `frame` | String | Frame style (for example, `"all"`, `"none"`)
| `colsep` | String | Column separator rule (`"0"` or `"1"`)
| `rowsep` | String | Row separator rule (`"0"` or `"1"`)
| `cols` | Integer | Number of columns
|===

.Example
[source,json]
----
{
  "type": "table",
  "attrs": {
    "xml_id": "tbl-options",
    "title": "Available Options",
    "cols": 3
  },
  "content": [
    { "type": "table_head", "content": [...] },
    { "type": "table_body", "content": [...] }
  ]
}
----

=== table_head

`Docbook::Mirror::Node::TableHead` +
PM_TYPE: `"table_head"`

The header section of a table. Contains `table_row` children. Mapped from
DocBook `<thead>`.

.Example
[source,json]
----
{
  "type": "table_head",
  "content": [
    {
      "type": "table_row",
      "content": [
        { "type": "table_cell", "content": [ { "type": "text", "text": "Name" } ] },
        { "type": "table_cell", "content": [ { "type": "text", "text": "Type" } ] },
        { "type": "table_cell", "content": [ { "type": "text", "text": "Description" } ] }
      ]
    }
  ]
}
----

=== table_body

`Docbook::Mirror::Node::TableBody` +
PM_TYPE: `"table_body"`

The body section of a table. Contains `table_row` children. Mapped from
DocBook `<tbody>`.

.Example
[source,json]
----
{
  "type": "table_body",
  "content": [
    {
      "type": "table_row",
      "content": [
        { "type": "table_cell", "content": [ { "type": "text", "text": "verbose" } ] },
        { "type": "table_cell", "content": [ { "type": "text", "text": "Boolean" } ] },
        { "type": "table_cell", "content": [ { "type": "text", "text": "Enable verbose output" } ] }
      ]
    }
  ]
}
----

=== table_row

`Docbook::Mirror::Node::TableRow` +
PM_TYPE: `"table_row"`

A single row in a table head or table body. Contains `table_cell` children.

.Example
[source,json]
----
{
  "type": "table_row",
  "content": [
    { "type": "table_cell", "content": [ { "type": "text", "text": "Cell 1" } ] },
    { "type": "table_cell", "content": [ { "type": "text", "text": "Cell 2" } ] }
  ]
}
----

=== table_cell

`Docbook::Mirror::Node::TableCell` +
PM_TYPE: `"table_cell"`

A single cell within a table row. Mapped from DocBook `<entry>`. Supports
DocBook-style column spanning (`namest`/`nameend`) and row spanning
(`morerows`), as well as alignment attributes.

.Attributes

[cols="1,1,3"]
|===
| Attribute | Type | Description
| `align` | String | Horizontal alignment (`"left"`, `"center"`, `"right"`)
| `valign` | String | Vertical alignment (`"top"`, `"middle"`, `"bottom"`)
| `namest` | String | Column name where spanning starts
| `nameend` | String | Column name where spanning ends
| `morerows` | Integer | Number of additional rows this cell spans
|===

.Example (simple cell)
[source,json]
----
{
  "type": "table_cell",
  "content": [ { "type": "text", "text": "Cell content" } ]
}
----

.Example (spanning cell)
[source,json]
----
{
  "type": "table_cell",
  "attrs": {
    "namest": "col1",
    "nameend": "col3",
    "morerows": 1,
    "align": "center"
  },
  "content": [ { "type": "text", "text": "Wide cell" } ]
}
----

== Admonitions

=== admonition

`Docbook::Mirror::Node::Admonition` +
PM_TYPE: `"admonition"`

A callout or notice block. Mapped from DocBook `<note>`, `<warning>`,
`<tip>`, `<caution>`, `<important>`, and `<danger>`. The specific admonition
type is carried in the `admonition_type` attribute rather than using separate
node types for each kind.

.Attributes

[cols="1,1,3"]
|===
| Attribute | Type | Description
| `admonition_type` | String | One of `"note"`, `"warning"`, `"caution"`, `"important"`, `"tip"`, `"danger"`
|===

.Example
[source,json]
----
{
  "type": "admonition",
  "attrs": { "admonition_type": "warning" },
  "content": [
    {
      "type": "paragraph",
      "content": [ { "type": "text", "text": "Do not delete this file." } ]
    }
  ]
}
----

`Node` (generic) +
PM_TYPE: `"annotation"`

An annotation element providing an explanatory note. Mapped from DocBook
`<annotation>`. Annotations carry an `xml_id` for cross-referencing and contain
inline content (typically paragraphs). The frontend renders annotations as
interactive markers with hover/click popups.

.Attributes

[cols="1,1,3"]
|===
| Attribute | Type | Description
| `xml_id` | String | XML identifier of the annotation
|===

.Example
[source,json]
----
{
  "type": "annotation",
  "attrs": { "xml_id": "ann1" },
  "content": [
    {
      "type": "paragraph",
      "content": [ { "type": "text", "text": "This term was introduced in version 2.0." } ]
    }
  ]
}
----

== Callouts

Callouts provide a mechanism for annotating code blocks with numbered markers
that link to descriptions in a `calloutlist`. The `<co>` elements are extracted
from code blocks as marker metadata (see `code_block` above), and the
`<calloutlist>` / `<callout>` elements form a corresponding numbered list.

=== calloutlist

`Node::CalloutList` +
PM_TYPE: `"calloutlist"`

A numbered list of callout descriptions. Mapped from DocBook `<calloutlist>`.
Contains `callout` children.

.Attributes

[cols="1,1,3"]
|===
| Attribute | Type | Description
| `xml_id` | String | XML identifier
| `title` | String | Optional title
|===

.Example
[source,json]
----
{
  "type": "calloutlist",
  "attrs": { "title": "Line annotations" },
  "content": [
    {
      "type": "callout",
      "attrs": { "arearefs": "co1" },
      "content": [
        {
          "type": "paragraph",
          "content": [ { "type": "text", "text": "The root element must be book or article." } ]
        }
      ]
    }
  ]
}
----

=== callout

`Node::Callout` +
PM_TYPE: `"callout"`

A single callout description referencing one or more `<co>` markers via
`arearefs`. Mapped from DocBook `<callout>`.

.Attributes

[cols="1,1,3"]
|===
| Attribute | Type | Description
| `xml_id` | String | XML identifier
| `arearefs` | String | Space-separated IDs of the `<co>` markers this callout describes
|===

== Glossary

Glossary entries use a dedicated set of node types to preserve the DocBook
glossary structure with terms, definitions, and cross-references.

=== glossary

`Node::Glossary` +
PM_TYPE: `"glossary"`

A glossary section. Mapped from DocBook `<glossary>`. Contains `gloss_entry`
children. When the `sort_glossary` option is enabled, entries are sorted
alphabetically by term.

.Attributes

[cols="1,1,3"]
|===
| Attribute | Type | Description
| `xml_id` | String | XML identifier
| `title` | String | Optional glossary title
|===

=== gloss_entry

`Node::GlossEntry` +
PM_TYPE: `"gloss_entry"`

A single glossary entry. Mapped from DocBook `<glossentry>`. Contains a
`gloss_term`, a `gloss_def`, and optional `gloss_see` / `gloss_see_also`
cross-references.

.Attributes

[cols="1,1,3"]
|===
| Attribute | Type | Description
| `xml_id` | String | XML identifier
|===

=== gloss_term

`Node::GlossTerm` +
PM_TYPE: `"gloss_term"`

The term being defined. Contains inline text content.

.Example
[source,json]
----
{
  "type": "gloss_term",
  "content": [ { "type": "text", "text": "XML" } ]
}
----

=== gloss_def

`Node::GlossDef` +
PM_TYPE: `"gloss_def"`

The definition of a glossary term. Contains block-level children (paragraphs,
lists, etc.).

=== gloss_see

`Node::GlossSee` +
PM_TYPE: `"gloss_see"`

A "see also" cross-reference to another glossary term. Mapped from DocBook
`<glosssee>`.

.Attributes

[cols="1,1,3"]
|===
| Attribute | Type | Description
| `otherterm` | String | The `xml:id` of the referenced term
|===

=== gloss_see_also

`Node::GlossSeeAlso` +
PM_TYPE: `"gloss_see_also"`

An additional "see also" cross-reference. Mapped from DocBook `<glossseealso>`.

.Attributes

[cols="1,1,3"]
|===
| Attribute | Type | Description
| `otherterm` | String | The `xml:id` of the referenced term
|===
