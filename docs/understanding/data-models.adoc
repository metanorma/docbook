# Data Models

The gem uses two model namespaces: the DocbookMirror nodes and marks
for content representation, and the Models namespace for SPA metadata.
All model classes inherit from `Lutaml::Model::Serializable`, which
provides XML and JSON serialization through a declarative DSL.

== DocbookMirror models (`Docbook::Mirror`)

The DocbookMirror system transforms DocBook element objects into a
ProseMirror-compatible JSON tree. The `Mirror::Transformer` walks the
element tree and produces typed nodes and marks.

=== Mirror::Node types

The transformer produces a tree of node types including:

* `Mirror::Node::Document` -- The root node containing all content
* `Mirror::Node::Block` -- Typed block nodes (paragraph, heading, code_block,
  blockquote, image, list, table, admonition, footnote, etc.)
* `Mirror::Node::Text` -- Text content with optional marks
* `Node::CalloutList` / `Node::Callout` -- Callout annotation lists for code
* `Node::Glossary` / `Node::GlossEntry` / `Node::GlossTerm` / `Node::GlossDef` --
  Glossary entries with optional alphabetical sorting
* `Node::GlossSee` / `Node::GlossSeeAlso` -- Glossary cross-references
* `Node::Sidebar` -- Sidebar blocks with optional title
* Annotation, Q&A (qandaset/qandaentry/question/answer) nodes for
  interactive content

=== Mirror::Mark types

Inline annotations applied to text nodes:

* `emphasis` -- Emphasized text
* `strong` -- Bold text
* `code` -- Inline code
* `link` -- Hyperlink with href
* `xref` -- Internal cross-reference
* `citation` -- Bibliographic citation

=== TOC and Numbering

The `SectionData` class represents the table of contents tree. Each
entry has an `id`, `title`, `type` (part, chapter, section, appendix,
etc.), an optional `number`, and `children` (an array of nested
`SectionData` instances).

`NumberingEntry` maps a section ID to its formatted number string.
The `NumberingBuilder` assigns numbers using different schemes:
Roman numerals for parts (I, II, III), Arabic numerals for chapters
(1, 2, 3), hierarchical dotted numbers for sections (1.1, 1.2.3),
and uppercase letters for appendices (A, B, C).

=== IndexTerm and IndexGroup

`IndexTerm` represents a single index entry with `primary`, `secondary`,
and `tertiary` term text, plus the `section_id` and `section_title`
where the term appears. `IndexGroup` groups terms under a `letter`
heading. The `Index` model holds a `title` and an array of `IndexGroup`
instances.

== Models namespace (`Docbook::Models`)

Defined in `lib/docbook/models/`, these classes represent the full
document structure as consumed by the Vue SPA. They include additional
metadata beyond what the output data models carry.

=== DocumentRoot

The top-level model for the complete document. Contains `title`,
`metadata` (a `DocumentMetadata` instance), `toc` (an array of
`TocNode`), `sections` (an array of `SectionRoot`), `index` (an array
of `IndexEntry`), `numbering` (an array of `SectionNumber`), and a
`reading_position`. Also includes `generator` and `generated_at`
strings for debugging.

=== DocumentMetadata

Stores document-level information: `author`, `author_name`, `title`,
`subtitle`, `productname`, `version`, `pubdate`, `release_info`, and
`abstract`. All fields are strings.

=== TocNode

A recursive tree node for the table of contents. Each node has an `id`,
`title`, `type`, `number`, optional `page` (for PDF navigation), and
`children` (an array of nested `TocNode` instances).

=== SectionRoot

Represents a section in the pre-computed document structure. Contains
`id`, `type`, `title`, `number`, and `children` (nested `SectionRoot`
instances). For reference entries, it also holds a `refname` string.

=== SectionNumber

A simple mapping of section `id` to its computed `number` string and
`type`. Used by the Vue SPA to display section numbers in the UI.

=== IndexEntry

Represents an index term with `primary`, `secondary`, `tertiary`,
`see_also`, `section_id`, `section_title`, and a `sort_key` for
alphabetical ordering.

=== ReadingPosition

Tracks the user's reading progress. Contains `section_id`,
`percentage`, `last_read_at`, `reading_mode`, and `scroll_position`.
This data is persisted in the Vue SPA via localStorage.

== Serialization

Both model namespaces serialize to JSON for the Vue SPA. The DocbookMirror
nodes and marks produce the content tree that `MirrorRenderer` renders in
the frontend. The `Models` namespace provides additional metadata like
TOC, numbering, and index data. The `Output::SinglePage` class combines
all of these into a single self-contained HTML file.
