# CLI Options

The `docbook` command-line tool is built on Thor. It provides six
primary commands plus a development-only `roundtrip` command.

== Global behavior

* The executable is `docbook` (installed to `exe/`).
* All commands that accept `INPUT` take a file path argument.
* The tool exits with code 0 on success and code 1 on failure.
* Error messages are written to stderr.

== build

Build an interactive HTML reader from DocBook XML. Produces a self-contained
file that works offline.

....
docbook build INPUT -o OUTPUT [options]
....

[cols="1,1,1,1,3"]
|===
| Option | Alias | Default | Description

| `--output PATH`
| `-o`
| _(required)_
| Output HTML file path.

| `--xinclude`
| _(none)_
| `true`
| Resolve XInclude directives before processing.

| `--image-search-dir DIR [...]`
| _(none)_
| XML file directory
| Additional directories to search for image files.

| `--image-strategy STRATEGY`
| _(none)_
| `data_url`
| Image handling: `data_url` (Base64 embed), `file_url` (absolute path), or `relative` (web-friendly).

| `--title TITLE`
| _(none)_
| Derived from document
| Page title shown in the browser tab.
|===

=== Examples

....
# Basic usage
docbook build book.xml -o book.html

# Custom image paths
docbook build book.xml -o book.html --image-search-dir media/ images/

# For web hosting
docbook build book.xml -o book.html --image-strategy relative

# Build the bundled demo
docbook demo -o demo.html
....

== convert

Convert DocBook XML to static HTML (no JavaScript).

....
docbook convert INPUT [options]
....

[cols="1,1,1,1,3"]
|===
| Option | Alias | Default | Description

| `--output PATH`
| `-o`
| stdout
| Output file path (single-file mode) or directory path (directory mode).

| `--xinclude`
| _(none)_
| `true`
| Resolve XInclude directives before processing.

| `--output-type TYPE`
| _(none)_
| `single_file`
| Output type: `single_file` or `directory`.
|===

=== Single-file mode

....
docbook convert book.xml -o output.html
....

Produces a single `output.html` with all JSON data, CSS, and JavaScript
embedded inline.

=== Directory mode

....
docbook convert book.xml -o my-book --output-type directory
....

Creates a directory with `index.html`, `docbook.data.json`, and `assets/`.

== export

Export DocBook XML as DocbookMirror JSON (ProseMirror-compatible).

....
docbook export INPUT [options]
....

[cols="1,1,1,1,3"]
|===
| Option | Alias | Default | Description

| `--output PATH`
| `-o`
| stdout
| Output JSON file path.

| `--pretty`
| _(none)_
| `true`
| Pretty-print the JSON output.

| `--xinclude`
| _(none)_
| `true`
| Resolve XInclude directives before processing.
|===

=== Examples

....
docbook export book.xml -o book.json
docbook export book.xml --no-pretty
....

== validate

Validate DocBook XML against the RELAX NG schema.

....
docbook validate INPUT [options]
....

[cols="1,1,1,1,3"]
|===
| Option | Alias | Default | Description

| `--schema`
| _(none)_
| `true`
| Validate against DocBook 5 RELAX NG schema.

| `--wellformed`
| _(none)_
| `false`
| Check well-formedness only (skip schema validation).
|===

When `--wellformed` is used, only XML parsing errors are checked.
Otherwise, the document is also validated against the bundled DocBook 5
RELAX NG schema.

== format

Format and prettify a DocBook XML file.

....
docbook format INPUT [options]
....

[cols="1,1,1,1,3"]
|===
| Option | Alias | Default | Description

| `--output PATH`
| `-o`
| stdout
| Output file path. When omitted, formatted XML is written to
  standard output.

| `--xinclude`
| _(none)_
| `false`
| Resolve XInclude directives before formatting.
|===

== demo

Build a demo reader from the bundled DocBook sample.

....
docbook demo [options]
....

[cols="1,1,1,1,3"]
|===
| Option | Alias | Default | Description

| `--output PATH`
| `-o`
| `demo.html`
| Output HTML file path.
|===

== roundtrip

Round-trip test one or more DocBook XML files.

....
docbook roundtrip INPUT [INPUT...]
....

This command has no options. It parses each file, serializes it,
re-parses the output, and reports success or failure.

== Quick reference table

[cols="1,1,1"]
|===
| Command | Input | Output

| `build`
| File path
| Self-contained interactive HTML

| `convert`
| File path
| Static HTML (file or directory)

| `export`
| File path
| DocbookMirror JSON (file or stdout)

| `validate`
| File path
| Pass/fail status

| `format`
| File path
| Pretty-printed XML (file or stdout)

| `demo`
| _(none)_
| Interactive HTML from bundled sample

| `roundtrip`
| One or more file paths
| Pass/fail status per file
|===
