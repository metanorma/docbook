== build

Build an interactive HTML reader from a DocBook XML file. The output format is controlled by `--format` (default: `inline`).

* `inline` -- Single self-contained HTML file (default). Works offline.
* `dom` -- Single HTML with pre-rendered content for SEO. Works offline.
* `dist` -- Directory with external JSON files. Requires HTTP server.
* `paged` -- Directory with per-chapter HTML pages. Requires HTTP server.

=== Usage

[source,bash]
----
$ docbook build [INPUT] [options]
$ docbook build --demo [options]
----

=== Arguments

[cols="2,4"]
|===
| Argument | Description

| `INPUT` | Path to the DocBook XML file (required unless `--demo` is used)
|===

=== Options

[cols="3,1,4"]
|===
| Option | Default | Description

| `--output`, `-o` PATH | `<input>.html` or `demo.html` | Output file or directory path
| `--format` FORMAT | `inline` | Output format: `inline`, `dom`, `dist`, `paged`
| `--demo` | _(off)_ | Build a bundled demo: `xslTNG` (default) or `model-flow`
| `--xinclude` | `true` | Resolve XInclude elements before processing
| `--image-search-dir` DIR | XML file directory | Directories to search for image files (accepts multiple values)
| `--image-strategy` STRATEGY | `data_url` | Image handling: `data_url`, `file_url`, or `relative`
| `--title` TITLE | Derived from document | Page title shown in the browser tab
| `--sort-glossary` | `false` | Sort glossary entries alphabetically by term
|===

=== Demo mode

Pass `--demo` to build a reader from a bundled demo fixture. Two demos are available:

* `xslTNG` (default) -- DocBook XSLT 2.0 Stylesheet Reference Guide. A multi-chapter book with code listings, admonitions, tables, images, cross-references, and index terms.
* `model-flow` -- "The Art of the Model Flow." A stylistic essay on document transformation pipelines, written in the manner of Sun Tzu's Art of War.

[source,bash]
----
# Default demo (xslTNG)
$ docbook build --demo

# Named demo
$ docbook build --demo=xslTNG
$ docbook build --demo=model-flow
----

=== Image Strategies

The `--image-strategy` option controls how images are handled in the output:

[cols="2,5"]
|===
| Strategy | Description

| `data_url` | Images are Base64-encoded and embedded in the HTML. Produces a fully self-contained file that works offline. Recommended for local viewing and single-file distribution.

| `file_url` | Images are referenced by absolute file system paths. Suitable for local viewing where image files remain at their original locations.

| `relative` | Images use relative paths from the output HTML file. Best for web hosting -- deploy the HTML and image files together to a static server.
|===

=== Examples

==== Build from a file (output derived automatically)

[source,bash]
----
$ docbook build book.xml
Built /path/to/book.html
----

==== Build with explicit output path

[source,bash]
----
$ docbook build book.xml -o output.html
Built /path/to/output.html
----

==== Custom image directories

[source,bash]
----
$ docbook build book.xml --image-search-dir media/ images/
Built /path/to/book.html
----

==== For web hosting

[source,bash]
----
$ docbook build book.xml --image-strategy relative
Built /path/to/book.html
----

==== Custom title

[source,bash]
----
$ docbook build manual.xml --title "Product Manual v3.0"
Built /path/to/manual.html
----

==== Skip XInclude resolution

[source,bash]
----
$ docbook build article.xml --no-xinclude
Built /path/to/article.html
----

==== Build the bundled demo

[source,bash]
----
$ docbook build --demo
Built /path/to/demo.html

$ docbook build --demo=xslTNG -o /tmp/sample.html
Built /tmp/sample.html

$ docbook build --demo=model-flow -o /tmp/model-flow.html
Built /tmp/model-flow.html
----

=== What's in the output

The built HTML file is a complete interactive reader with:

* **Sidebar** -- Collapsible table of contents with section type badges
* **Search** -- Full-text search powered by FlexSearch (press `/` to open)
* **Themes** -- Day, Sepia, Night, and OLED themes with font options
* **Breadcrumb** -- Ancestor chain navigation bar
* **Content** -- All DocBook content rendered with proper typography

The file works offline because all CSS, JavaScript, and (with `data_url` strategy) images are embedded.
