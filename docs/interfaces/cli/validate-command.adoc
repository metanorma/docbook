== validate

Validate a DocBook XML file against the DocBook 5 RELAX NG schema. Also checks XML well-formedness.

=== Usage

[source,bash]
----
$ docbook validate INPUT [options]
----

=== Arguments

[cols="2,4"]
|===
| Argument | Description

| `INPUT` | Path to the DocBook XML file to validate (required)
|===

=== Options

[cols="3,1,4"]
|===
| Option | Default | Description

| `--schema` | `true` | Validate against DocBook 5 RELAX NG schema
| `--wellformed` | `false` | Check well-formedness only (skip schema validation)
|===

=== Behavior

The `validate` command performs two levels of validation:

. **Well-formedness check** -- Always performed. Parses the XML with Nokogiri and reports any syntax errors.
. **Schema validation** -- Performed by default. Validates against the bundled DocBook 5 RELAX NG schema.

The command automatically selects the appropriate schema:

* Documents with XInclude namespace (`http://www.w3.org/2001/XInclude`) use `docbookxi.rng`
* All other documents use `docbook.rng`

Use `--wellformed` to skip schema validation and check XML syntax only.

=== Exit Codes

[cols="1,4"]
|===
| Code | Condition

| `0` | The file is valid (well-formed and schema-compliant, or well-formed only with `--wellformed`)
| `1` | The file contains errors
|===

=== Examples

==== Validate against schema (default)

[source,bash]
----
$ docbook validate article.xml
article.xml: valid
----

==== Check well-formedness only

[source,bash]
----
$ docbook validate --wellformed article.xml
article.xml: valid
----

==== Invalid file

When the XML contains errors, each error is reported to stderr:

[source,bash]
----
$ docbook validate broken.xml
broken.xml: 6:14: FATAL: Opening and ending tag mismatch
$ echo $?
1
----

==== Schema validation error

[source,bash]
----
$ docbook validate invalid.xml
invalid.xml: Element 'badtag': This element is not expected. Expected is one of ...
$ echo $?
1
----

==== Validate multiple files in a script

[source,bash]
----
#!/bin/bash
set -e
for f in xml/*.xml; do
  docbook validate "$f"
done
echo "All files are valid"
----

==== Use in CI pipelines

[source,bash]
----
# GitHub Actions example
- name: Validate DocBook files
  run: |
    find . -name '*.xml' -exec docbook validate {} \;
----
