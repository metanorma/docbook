# HTML Generation

The `Output::SinglePage` class generates a self-contained HTML file by assembling the HTML shell, embedding all CSS and JavaScript inline, and injecting the DocbookMirror JSON data directly into the page.

== HTML Structure

The generated HTML file contains:

* A `<head>` section with fonts, Prism.js styles, Tailwind CSS, and the compiled app CSS all inlined.
* A `<div id="docbook-app">` mount point for the Vue SPA.
* An inline `<script>` block that defines `window.DOCBOOK_DATA` containing the DocbookMirror JSON, TOC, numbering, and index data.
* The compiled app JavaScript and Prism.js scripts inlined at the end of the body.

== Head Content

The `<head>` section includes:

* **Prism.js CSS** -- Syntax highlighting theme loaded from cdnjs.
* **Google Fonts** -- Inter (sans-serif), Merriweather (serif), and
  JetBrains Mono (monospace) loaded from Google Fonts CDN.
* **Tailwind CSS** -- The Tailwind CDN script with a custom
  configuration that sets `darkMode: 'class'` and registers the three
  font families.
* **Custom styles** -- Inline `<style>` block with TOC active states,
  bibliography entry indentation, and appendix border styling.
* **App CSS** -- The compiled `app.css` from `frontend/dist/` embedded in a `<style>` tag.

== Script Content

* **Prism.js** -- Core library and autoloader plugin from cdnjs for
  syntax highlighting of code blocks.
* **App JS** -- The compiled `app.iife.js` from `frontend/dist/` embedded in a `<script>` tag.

== Data Injection

The DocbookMirror JSON data (including TOC, numbering, and index) is embedded directly into the HTML as `window.DOCBOOK_DATA`. The Vue SPA's `documentStore` reads this data synchronously on load. The `MirrorRenderer` component then renders the DocbookMirror nodes into the interactive reader.

All data is embedded inline. The resulting HTML file is self-contained
and can be opened directly in a browser without a web server.
